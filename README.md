## Overview

Distribyte is a production-ready forecasting API and dashboard built for inventory management use cases. It trains a **Bidirectional LSTM with self-attention** on historical time series data and generates multi-step ahead forecasts with a clean REST API and interactive frontend.

**Live demo:** [distribyte.vercel.app](https://distribyte.vercel.app)

---

## Features

- **BiLSTM + Attention model** — stacked bidirectional LSTM layers with a self-attention mechanism for capturing long-range temporal dependencies
- **Multi-step autoregressive forecasting** — iteratively predicts n steps into the future using the model's own output as input
- **Model registry** — train and persist multiple models, each identified by a unique ID with metadata stored in JSON
- **MinMax scaling** — per-model feature and target scalers saved alongside model weights for consistent inference
- **REST API** — full CRUD for models, train and forecast endpoints with Pydantic validation
- **React dashboard** — time series visualization with historical + forecast overlay, model management UI, and demo mode

```
│                                                      │
│  Input (seq_len, n_features)                        │
│      → BiLSTM(64, return_sequences=True)            │

│      → Self-Attention                               │

│      → Dropout(0.2)                                 │
│      → LSTM(16)                                     │
│      → Dense(1)                                     │
└─────────────────────────────────────────────────────┘
```

## API Reference

### `POST /train`

Train a new forecasting model on your time series data.

**Request body:**
```json
{
  "time_series": [
    {"date": "2024-01-01", "sales": 4200, "inventory": 1800, "price": 29.99, "promotions": 0}
  ],
  "feature_columns": ["sales", "inventory", "price", "promotions"],
  "target_column": "sales",
  "date_column": "date",
  "sequence_length": 12,
  "forecast_horizon": 7
}
```

**Response:**
```json
{
  "model_id": "forecast_model_20240426142300",
  "feature_columns": ["sales", "inventory", "price", "promotions"],
  "target_column": "sales",
  "date_column": "date",
  "sequence_length": 12,
  "forecast_horizon": 7,
  "created_at": "2024-04-26T14:23:00.000000"
}
```

---

### `POST /forecast`

Generate multi-step forecasts using a trained model.

**Request body:**
```json
{
  "model_id": "forecast_model_20240426142300",
  "input_data": [...],
  "horizon": 14
}
```

**Response:**
```json
{
  "forecasts": [
    {"date": "2024-04-27", "sales": 4350.21},
    {"date": "2024-04-28", "sales": 4180.87}
  ]
}
```

---

### `GET /models`

List all trained models in the registry.

### `GET /models/{model_id}`

Get metadata for a specific model.

### `DELETE /models/{model_id}`

Remove a model and its associated files from disk.

## Quickstart

### 1. Clone and install

```bash
git clone https://github.com/Pshyam17/inventory.git
cd inventory
pip install fastapi uvicorn tensorflow scikit-learn pandas numpy joblib pydantic
```

### 2. Start the API

```bash
uvicorn "bilstm-attention-forecasting-api:app" --reload --host 0.0.0.0 --port 8000
```

API docs available at `http://localhost:8000/docs` (Swagger UI auto-generated by FastAPI).

### 3. Run the frontend

The dashboard is a single-page React app. Open `forecasting-login-page.html` directly in a browser, or point the API base URL in Settings to your running backend.

---

## Data Format

Your CSV or JSON input must include:

| Column | Type | Description |
|---|---|---|
| `date` | string | ISO date (YYYY-MM-DD) |
| `sales` | float | Target variable to forecast |
| `inventory` | float | Feature: current stock level |
| `price` | float | Feature: unit price |
| `promotions` | int | Feature: promotion flag (0/1) |

Column names are fully configurable via the `feature_columns`, `target_column`, and `date_column` parameters.

### Frontend (Vercel)

```bash

```

### Backend (Docker + Azure / Railway)

```dockerfile

COPY . .

CMD ["uvicorn", "bilstm-attention-forecasting-api:app", "--host", "0.0.0.0", "--port", "8000"]
```

```bash
docker build -t distribyte-api .
docker run -p 8000:8000 distribyte-api
```
